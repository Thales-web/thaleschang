## Overview

The Aside component renders a styled callout box to draw attention to important information. It is useful for tips, warnings, supplementary notes, and critical alerts that should stand out from the surrounding content.

Aside is auto-imported into every MDX file, so you can use it directly without an import statement.

## Variants

Aside supports four visual variants, each designed for a specific type of message. The variant determines the border color, background tint, and icon displayed alongside the content.

### Info (Default)

The `info` variant is the default. Use it for general supplementary information that adds context without urgency.

```mdx
<Aside variant="info" title="Additional context">
This endpoint returns paginated results. The default page size is 25 items.
You can adjust this using the `limit` query parameter.
</Aside>
```

<Aside variant="info" title="Additional context">
This endpoint returns paginated results. The default page size is 25 items.
You can adjust this using the `limit` query parameter.
</Aside>

### Tip

The `tip` variant highlights helpful suggestions and best practices. Use it when you want to share a recommendation that can improve the reader's workflow.

```mdx
<Aside variant="tip" title="Performance tip">
Enable response caching on frequently accessed endpoints to reduce latency.
Set the `Cache-Control` header to `max-age=300` for a five-minute cache window.
</Aside>
```

<Aside variant="tip" title="Performance tip">
Enable response caching on frequently accessed endpoints to reduce latency.
Set the `Cache-Control` header to `max-age=300` for a five-minute cache window.
</Aside>

### Caution

The `caution` variant warns about potential issues or behaviors that could cause unexpected results. Use it when the reader should proceed with awareness.

```mdx
<Aside variant="caution" title="Rate limiting">
The API enforces a rate limit of 100 requests per minute per API key.
Exceeding this limit will result in `429 Too Many Requests` responses
for the remainder of the rate window.
</Aside>
```

<Aside variant="caution" title="Rate limiting">
The API enforces a rate limit of 100 requests per minute per API key.
Exceeding this limit will result in `429 Too Many Requests` responses
for the remainder of the rate window.
</Aside>

### Danger

The `danger` variant signals critical warnings about destructive actions, security risks, or breaking changes. Use it sparingly for situations where ignoring the message could lead to data loss or security vulnerabilities.

```mdx
<Aside variant="danger" title="Destructive action">
The `DELETE /users/{id}` endpoint permanently removes the user account
and all associated data. This action cannot be undone. Always confirm
with the user before calling this endpoint in production.
</Aside>
```

<Aside variant="danger" title="Destructive action">
The `DELETE /users/{id}` endpoint permanently removes the user account
and all associated data. This action cannot be undone. Always confirm
with the user before calling this endpoint in production.
</Aside>

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `"info" \| "tip" \| "caution" \| "danger"` | `"info"` | Controls the visual style and icon of the aside |
| `title` | `string` | — | Optional heading text displayed at the top of the aside |

## Content Support

The Aside component supports full Markdown content inside its body, including:

- **Inline formatting** — Bold, italic, strikethrough, and inline code
- **Links** — Both internal and external links
- **Lists** — Ordered and unordered lists
- **Code blocks** — Fenced code blocks with syntax highlighting

### Example with Rich Content

```mdx
<Aside variant="tip" title="Configuration example">
To enable debug logging, add the following to your configuration file:

\`\`\`typescript
export const config = {
  debug: true,
  logLevel: "verbose",
};
\`\`\`

See the [configuration reference](/docs/reference/site-settings) for all available options.
</Aside>
```

<Aside variant="tip" title="Configuration example">
To enable debug logging, add the following to your configuration file:

```typescript
export const config = {
  debug: true,
  logLevel: "verbose",
};
```

See the [configuration reference](/docs/reference/site-settings) for all available options.
</Aside>

## Without a Title

You can omit the `title` prop to render an aside without a heading. The icon and colored border will still be displayed:

```mdx
<Aside variant="info">
All timestamps in API responses are in UTC format following the ISO 8601 standard.
</Aside>
```

<Aside variant="info">
All timestamps in API responses are in UTC format following the ISO 8601 standard.
</Aside>

## Usage Guidelines

### When to Use Aside

- To highlight prerequisites before a set of instructions
- To warn about common pitfalls or breaking changes
- To share tips that improve the reader's experience
- To call attention to security considerations

### When Not to Use Aside

- For primary instructional content (use regular prose instead)
- For every paragraph (overuse reduces the impact of callouts)
- For decorative purposes without meaningful content

### Choosing the Right Variant

| Situation | Variant |
|---|---|
| General background information | `info` |
| Best practices and recommendations | `tip` |
| Potential issues or gotchas | `caution` |
| Data loss, security risks, or breaking changes | `danger` |

## Accessibility

The Aside component uses appropriate ARIA roles and attributes:

- The aside is rendered as a `<div>` with `role="note"` for screen readers
- The title, when provided, is rendered as a heading for document outline navigation
- Color is never the sole indicator of meaning; each variant also includes a distinct icon
- Content inside the aside maintains proper heading hierarchy
